Table with sticky header & first column

The table will switch between two modes

  1. If the users device is wide enough to show all columns (Desktop) then the header will stick to the top of the viewport and the tables height value is auto.
  2. If the users device is to narrow to show all columns (Mobile) then the first column will stick to the left and the table will have a static height with a vertical scrollbar.

Why the fixed table-height on mobile?

The desktop view works perfectly so why the fixed table height on mobile devices?

Well unfortunately the css property "overflow" that prevents the table columns to break the page's width is breaking the header's stickyness. A sticky element will stick to it's closest parent which has the property "overflow" set, which usualy would be the document body tag. But in this case the header will stick to the top of the table-wrapper (.table) because it has the property "overflow" set.
